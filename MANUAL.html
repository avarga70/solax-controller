Solar Controller

GoodWe 10K ET  ·  HA Daemon  ·  Czech OTE Spot Prices  ·  Usage Manual

Overview

solar_controller.py is a daemon that runs every few minutes (default 5) and optimises your GoodWe 10K ET inverter based on:

  • Czech spot electricity prices from the local MySQL EL_dailypr table (CZK/MWh)
  • Current battery SOC and inverter state via the goodwe UDP library
  • Weather-based PV generation forecast from forecast.solar (free, no account needed) — falls back to historical PV_run patterns when unavailable
  • Planned grid outage schedule from planned_outages.json

It never exports battery energy to the grid. It controls the inverter operation mode and the battery minimum discharge SOC.

TEST MODE  Runs the full decision logic and logs everything but sends no commands to the inverter. Safe to leave running for days before going live.

Quick Start

  1. Install dependencies (once per node): 
    python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
  2. Configure: 
    cp config.env.example config.env
# Edit config.env — set INVERTER_IP, MYSQL_*, NODE_NAME, TEST_MODE=1
  3. Test a single cycle: 
    .venv/bin/python3 solar_controller.py --test --once
  4. Run as a service: 
    sudo systemctl start  solar-controller
sudo systemctl enable solar-controller   # start on boot
sudo systemctl status solar-controller
  5. Go live — set in config.env on each node: 
    TEST_MODE=0
sudo systemctl restart solar-controller

Arbitrage Profitability Gate

Grid charging is skipped if the battery cannot earn back the round-trip cost at the best upcoming sell price: min_sell = (buy + DEVIATION_FEE) / BATTERY_EFFICIENCY

Decision Rules

Rules are evaluated in priority order once per cycle. The first matching rule wins.

0 Planned Outage Inside window → GENERAL, min SOC = OUTAGE_PRE_CHARGE_SOC
Approaching → ECO_CHARGE, min SOC = OUTAGE_PRE_CHARGE_SOC (lead time = Wh needed ÷ CHARGE_RATE_W, max OUTAGE_LEAD_HOURS)
1 Emergency SOC Battery at or below MIN_SOC_FLOOR → ECO_CHARGE (fast grid charge)
1.5 Negative Price Spot price ≤ 0 → ECO_CHARGE (absorb free electricity into battery)
2 Cheap Price Price ≤ cheap threshold & battery not full & arbitrage profitable & PV won't fill it → ECO_CHARGE, min SOC = CHARGE_SOC_LIMIT
Arbitrage gate: charging is skipped if max upcoming sell price < (buy_price + DEVIATION_FEE) / BATTERY_EFFICIENCY
2.5 Night Prep-Charge Overnight only: expected load before the next cheap window exceeds battery headroom → ECO_CHARGE at the cheapest available overnight hour. Ensures the battery arrives at the morning peak with sufficient charge.

Oscillation guard: skipped if Rule 3 discharge is already profitable (buying to 98% only to immediately discharge would waste round-trip losses).
Opportunity-cost gate: skipped when deeply negative midday prices are upcoming — preserving headroom to absorb free electricity is worth more than the overnight pre-charge saving.
3 Expensive Now Price ≥ expensive threshold & hour ≥ discharge_start → GENERAL, min SOC = MIN_SOC_FLOOR (battery covers house load)
Floor raised to preserve_min_soc if expensive morning peak hours arrive before the next cheap window; lowered back when a cheaper overnight pre-charge is coming — discharge freely now.
If a cheap recharge comes before discharge_start: discharge now, recharge, discharge again.
Continuous expensive block (no cheap gap): discharge straight through.
4–5 Peak Preservation / End-of-Day Rule 4: Expensive hours coming later today with PV running — raise floor to preserve SOC for the upcoming peak.

Rule 5: End-of-day window (hour ≥ eod_hour, not yet night). eod_hour is computed dynamically each cycle as the last PV hour with > 500 Wh + 1 (EOD_HOUR config is a fallback only — used only when no forecast is available).
• Before discharge_start → GENERAL, min SOC = max(ECO_MIN_SOC, current SOC) — holds at current level, preventing Rule 2 from topping up only for Rule 5 to drain back down (charge/discharge oscillation guard).
• At/after discharge_start → GENERAL, floor by morning outlook.
6 Overnight Hold / Drain Expensive peak arriving before the next cheap window (e.g. h06–h08 peak, cheap h10+):
Hold phase (hour < drain_start): GENERAL, min SOC = hold_floor — grid covers house load until the drain window opens.
Drain phase (hour ≥ drain_start): GENERAL, min SOC = NIGHT_MIN_SOC — battery drains to cover morning peak load until the cheap window.
With SMART_DISCHARGE=1: hold_floor is lowered from CHARGE_SOC_LIMIT to the minimum SOC needed at drain_start (+3% buffer), so the battery can discharge profitably at pre-peak prices rather than sitting idle at 98%.
— Default GENERAL, min SOC = preserve_min_soc (preserves charge for morning peak)
discharge_start is the earliest hour the battery should freely discharge. With SMART_DISCHARGE=1 it is the beginning of the block of hours where discharging beats buying at the cheapest upcoming recharge price, ranked by effective_price × expected_load. Without it, it is simply the highest-price hour the battery can cover.

Planned Outages (planned_outages.json)

Edit the file — no service restart needed, it is re-read every cycle:

[
  {
    "start": "2026-03-05 08:00",
    "end":   "2026-03-05 14:00",
    "note":  "planned grid maintenance"
  }
]

Multiple entries supported. Past entries are ignored automatically. Lead time is calculated dynamically from the current SOC each cycle.

Inverter diagnostics

Lists all inverter settings keys — use this to confirm the correct key for the battery min SOC setting (look for soc, dod, or protect).

High Availability Two-Node

Both nodes run the daemon. Only the LEADER connects to the inverter and applies commands. Leader election uses a MySQL lease table (solar_leader_lease).

Nodeconfig.env settings
node1HA_ENABLED=1   NODE_NAME=node1
node2HA_ENABLED=1   NODE_NAME=node2

If the leader fails, the standby takes over within 2 × LOOP_INTERVAL_MINUTES + 30 s (~40 s with the default 5-minute cycle). The inverter's last settings remain active in the meantime.

Deploy changes to both nodes

./deploy.sh --restart
